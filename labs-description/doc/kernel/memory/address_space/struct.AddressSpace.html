Expand description

Структура для работы с виртуальным адресным пространством.

Fields

info: AtomicInfo

Статистика аллокатора общего назначения.

kind: Kind

Тип виртуального адресного пространства для логирования:

  • Kind::Invalid — некорректное;
  • Kind::Base — базовое;
  • Kind::Process — относящееся к некоторому процессу.
mapping: Mapping

Структура для управления отображением виртуальных страниц в физические фреймы.

page_allocator: PageAllocator

Аллокатор виртуальных страниц.

pid: Option<Pid>

Процесс, к которому относится данное адресное пространство. Содержит None, если:

  • у процесса ещё нет Pid или
  • это адресное пространство не относится ни к какому процессу.

pub unsafe fn map_page_to_frame(
    &mut self,
    page: Page,
    frame: Frame,
    flags: PageTableFlags
) -> Result<()>

Отображает заданную виртуальную страницу page на заданный физический фрейм frame с указанными флагами доступа flags.

Если page уже была отображена, то старое отображение удаляется, если только при этом не произойдёт замена прав доступа к странице с “только для ядра” на “доступно в пользовательском пространстве” — PageTableFlags::USER_ACCESSIBLE. В случае попытки такой замены отображение не меняется, а возвращается ошибка Error::PermissionDenied. Если же отображение page поменялось, старый физический фрейм освобождается, если на него не осталось других ссылок.

Safety

Вызывающий код должен гарантировать, что инварианты управления памятью в Rust’е не будут нарушены. В частности, не осталось ссылок, которые ведут в страницу page.

pub(crate) fn check_permission<T>(
    &mut self,
    block: Block<Virt>,
    flags: PageTableFlags
) -> Result<&'static [T]>

Проверяет доступность блока виртуальной памяти block на чтение и соответствие заданным флагам доступа flags. Возвращает иммутабельный срез типа T, расположенный в блоке block.

Это позволяет объединить в одно действие проверку доступа и упростить последующий доступ к памяти, которую процесс пользователя указал ядру в системном вызове.

    new_block: Block<Page>
) -> Result<()>

Перемещает отображение физических фреймов из old_block в new_block. Если изначально new_block содержал отображённые страницы, их отображение удаляется.
